Intro a Javascript.

En esta clase vamos a ver un poco de como funciona Javascript, que son variables y constantes, la diferencia entre las palabras reservadas let, var y const, tipos de datos, estructuras secuenciales, bloques de codigo y estructuras de seleccion.

Variables

Las variables son espacios de almacenamiento a los cuales les asignamos un nombre para acceder a esos espacios despues, estas pueden variar durante el tiempo de ejecucion de la pagina.
Un ejemplo seria: podemos tener una variable donde guardamos el numero de veces que el usuario ha ganado premios en nuestra plataforma de rifas.
Las variables las declaramos usando las palabras var o let, la diferencia entre estas dos palabras es el scope, es decir desde que partes del codigo podemos acceder a esas variables, las que declaramos con var pueden usarse dentro de la funcion, mientras que las que declaramos usando let solo pueden usarse dentro del bloque, es decir dentro del par de llaves (marcado por {}) donde se encuentran.

Constantes

Al igual que las variables, las constantes son un espacio en memoria que tiene un identificador, pero que no vamos a modificar, es decir, valores que permaneceran igual durante todo el ciclo de vida de nuestro codigo, por convencion se escriben en mayusculas.

Tenemos diferentes tipos de datos, estos tienen diferentes propiedades y los usamos para situaciones especificas, en javascript el tipo de variable puede cambiar con el tiempo, asi que debemos tener cuidado con las operaciones que hacemos.
Estos son:

  • String: Son lineas de texto, un string puede ser un nombre, una propiedad, un texto, o una serie de caracteres.
  • Numbers: Como lo dice su nombre, estos pueden escribirse enteros o decimales usando un punto.
  • Booleans: Solo tienen dos valores posibles, verdadero o falso.
  • Arrays: Son colecciones de elementos, se escriben con llaves cuadradas [].
  • Objects: Son elementos con propiedades y funciones, se escriben con llaves {}.

Estructuras secuenciales

Una estructura secuencial es aquella en la que cada paso ocurre despues de otro en secuencia, muchos de estos pasos dependen de pasos que ocurrieron anteriormente.
Podemos pensarlo como una serie de pasos que deben ocurrir para llegar a un objetivo final.

Bloques de codigo

Es codigo que tenemos agrupado entre llaves, esto es para delimitar o para ordenar nuestro codigo, podemos crear bloques en cualquier momento como por ejemplo creando una funcion que ejecutara operaciones y nos devuelve un valor, o un objeto con propiedades especificas.

Estructuras de seleccion

Con estas podemos tomar decidiones de que camino deberia tomar nuestra aplicacion, por ejemplo que hacer si tenemos o no un usuario valido, o si el numero de estudiantes es mayor a 20.

  • if: un if es una forma de verificar si se cumple una condicion, como lo aprendimos en la clase de logica y argumentacion: si a entonces b, b solamente va a ocurrir en caso de que se cumpla la condicion a.
  • if/else: el if/else funciona igual que el if pero nos permite decirle que hacer cuando la condicion no se cumple.
  • switch: nos permite dar casos o condiciones especificas y que hacer cuando se cumplen esas condiciones.
